# Engineering Rules

- Prefer small, reviewable changes.
- Follow existing patterns before creating new abstractions.
- Do not rewrite unrelated areas while solving a local problem.
- Keep functions and modules focused.
- Make assumptions explicit.
- When changing behavior, update tests and docs where appropriate.
- If a command or verification step cannot be run, state that clearly.
- Favor clarity over cleverness.
- Prefer deterministic solutions over prompt-only guidance when guardrails are needed.
