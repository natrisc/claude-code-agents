---

# Workflow

1. define consumer and use case
2. define request and response shapes
3. define validation rules
4. define error cases
5. define auth requirements
6. assess backward compatibility
7. identify test cases

## Output

- endpoint or interface
- request schema
- response schema
- validation
- errors
- auth
- compatibility notes
- tests
