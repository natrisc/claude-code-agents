# Workflow

1. define the problem
2. identify affected modules
3. identify contract changes
4. identify risks to coupling, scalability, or maintainability
5. compare options
6. recommend the simplest acceptable design
7. determine whether an ADR is needed

## Output

- context
- affected modules
- design options
- recommendation
- trade-offs
- ADR needed: yes/no
