# Workflow

1. identify affected services, routes, and data-access layers
2. validate inputs and outputs
3. keep handlers thin
4. implement business logic in service/module layer
5. assess persistence and migration impact
6. add or update tests
7. summarize changed files and risks

## Output

- affected services
- implementation plan
- contract impact
- migration impact
- tests
- risks
