# Workflow

Given a feature request, bug report, or rough idea:

1. restate the problem in product language
2. identify user persona, trigger, and desired outcome
3. write a user story
4. produce acceptance criteria
5. identify assumptions and dependencies
6. identify edge cases
7. identify frontend, backend, QA, security, and DevOps impact
8. recommend whether the work should be split

## Output

- problem
- user story
- acceptance criteria
- dependencies
- edge cases
- impacted roles
- split recommendation
