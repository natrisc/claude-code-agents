# Workflow

1. identify affected screens and components
2. define state and data flow
3. check reuse of existing UI primitives
4. implement minimal UI change
5. update interaction and accessibility coverage
6. summarize changed files and risks

## Output

- affected components
- implementation plan
- accessibility considerations
- tests
- risks
