# Project Context Change Log

Each entry records a formal change to `project_context.md`.

---

## Template

```
## CL-NNN

- **Date**: YYYY-MM-DD
- **Version from**: X.Y
- **Version to**: X.Z
- **Change type**: addition | removal | clarification | correction
- **Changed by**: product-owner
- **Summary**: One-line description
- **Rationale**: Why the change was needed
- **Impact assessment**: Which downstream artifacts are affected
- **Required follow-up actions**:
  - [ ] Action 1
- **Invalidations**:
  - analysis/requirements.md
  - architecture/architecture.md
```
